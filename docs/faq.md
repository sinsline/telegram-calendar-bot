## 🤔 Общие вопросы

### Что это за бот и для чего он нужен?
Telegram Calendar Bot - это умный помощник для управления календарем через Telegram. Он может:
- Распознавать голосовые сообщения и создавать события
- Извлекать даты из изображений (визитки, расписания)
- Парсить события из текста на русском, немецком и английском языках
- Синхронизироваться с Google Calendar

### Является ли бот бесплатным?
Да, исходный код бота полностью бесплатен и открыт (MIT License). Однако могут потребоваться затраты на:
- VPS/облачный хостинг для развертывания
- Google Cloud Platform (при превышении бесплатных лимитов)

### Какие данные собирает бот?
Бот собирает только данные, необходимые для работы:
- Сообщения пользователей для парсинга событий
- Временные файлы (аудио, изображения) - удаляются после обработки
- Логи действий (без личных данных, только события системы)

Бот НЕ хранит:
- Личную информацию пользователей
- Содержимое календарей
- Токены доступа к другим сервисам

### Поддерживает ли бот групповые чаты?
В текущей версии бот работает только в личных сообщениях. Поддержка групп запланирована в будущих релизах.

## 🔧 Установка и настройка

### Какие системные требования?
**Минимальные:**
- 1 CPU ядро, 512MB RAM
- 2GB свободного места на диске
- Docker или Python 3.11+

**Рекомендуемые (продакшн):**
- 2+ CPU ядра, 1GB+ RAM  
- 10GB свободного места
- Ubuntu 20.04+ или аналогичная ОС

### Как получить Telegram Bot Token?
1. Напишите [@BotFather](https://t.me/botfather) в Telegram
2. Отправьте команду `/newbot`
3. Выберите имя и username для бота
4. Скопируйте полученный токен
5. Добавьте токен в `.env` файл:
   ```env
   TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
   ```

### Обязательна ли интеграция с Google Calendar?
Нет, Google Calendar интеграция опциональна. Без неё бот будет:
- Парсить события из текста/голоса/изображений
- Показывать структурированную информацию
- Сохранять события в локальной базе (если настроена)

С Google Calendar интеграцией добавляется:
- Синхронизация с реальным календарем
- Напоминания и уведомления
- Доступ к событиям с других устройств

## 📅 Google Calendar

### Какой тип Google аккаунта лучше использовать?
**Service Account (рекомендуется):**
- ✅ Стабильная авторизация без повторной аутентификации
- ✅ Хорошо для серверных приложений
- ✅ Простая настройка

**OAuth2:**
- ✅ Использует ваш личный Google аккаунт
- ❌ Требует периодической повторной авторизации
- ❌ Более сложная настройка

### Ошибка "Calendar not found" - что делать?
1. **Проверьте Calendar ID:**
   ```bash
   make config
   # Убедитесь что GOOGLE_CALENDAR_ID правильный
   ```

2. **Для Service Account** - предоставьте доступ:
   - Откройте Google Calendar
   - Найдите ваш календарь в левом меню
   - Settings → Share with specific people
   - Добавьте email Service Account (из credentials.json)
   - Выберите "Make changes to events"

3. **Проверьте права доступа:**
   ```bash
   # Тестирование подключения
   python test_calendar.py
   ```

### Можно ли использовать корпоративный G Suite?
Да, но потребуется дополнительная настройка:
1. **Domain-wide delegation** в G Suite Admin Console
2. **Service Account** с правами на календари пользователей
3. Указание `subject` (пользователя) при создании credentials

Подробнее: [docs/google-calendar-setup.md](google-calendar-setup.md)

### Лимиты Google Calendar API
- **Бесплатно**: 1,000,000 запросов в день
- **Квоты**: 100 запросов в 100 секунд на пользователя
- **Burst**: до 10 запросов в секунду

При превышении лимитов бот автоматически будет делать паузы между запросами.

## 🤖 Использование бота

### Какие форматы дат понимает бот?
**Абсолютные даты:**
```
15 января 2024
2024-01-15  
15/01/2024
15.01.2024
```

**Относительные даты:**
```
сегодня, завтра, послезавтра
в понедельник, во вторник, в пятницу
через неделю, через месяц
на следующей неделе
```

**Время:**
```
в 15:00, в 15-00, в 3 часа дня
в половине четвертого, в четверть восьмого  
утром, днем, вечером, ночью
```

### Примеры команд для создания событий

**Простые события:**
```
встреча с клиентом завтра в 15:00
звонок маме в пятницу в 19:30
дентист 25 декабря в 10 утра
```

**Сложные события:**
```
встреча с командой завтра с 14:00 до 16:00 в конференц-зале
презентация проекта в понедельник в 10 утра, длительность 2 часа
звонок с партнерами из Германии во вторник в 15:00 по берлинскому времени
```

**Периодические события:**
```
тренировка каждый вторник в 18:00
планерка каждый понедельник в 09:30
день рождения 15 марта каждый год
```

### Как работает распознавание голоса?
Бот использует **OpenAI Whisper** для распознавания речи:
- Поддерживает 99+ языков
- Работает оффлайн (после скачивания модели)
- Высокая точность даже с акцентом
- Максимальный размер файла: 25MB

**Поддерживаемые форматы аудио:**
- OGG (голосовые сообщения Telegram)
- MP3, WAV, M4A, FLAC

### Как работает распознавание изображений?
Бот использует **Tesseract OCR**:
- Извлекает текст с фотографий визиток, расписаний
- Поддерживает русский, английский, немецкий языки
- Автоматически улучшает качество изображения
- Работает с JPEG, PNG, WebP, TIFF, BMP

**Советы для лучшего распознавания:**
- Хорошее освещение и четкий текст
- Минимальный угол наклона камеры
- Высокое разрешение (но не более 4K)

### Что делать если бот не понял событие?
1. **Упростите формулировку:**
   ```
   Вместо: "надо бы встретиться с Андреем как-нибудь завтра после обеда"
   Напишите: "встреча с Андреем завтра в 15:00"
   ```

2. **Укажите точное время:**
   ```
   Вместо: "звонок маме вечером"  
   Напишите: "звонок маме в 19:00"
   ```

3. **Разбейте сложное событие:**
   ```
   Вместо: "встреча с клиентом и презентация проекта завтра утром"
   Создайте два события:
   - "встреча с клиентом завтра в 10:00"
   - "презентация проекта завтра в 11:00"
   ```

## ⚠️ Проблемы и ошибки

### Бот не отвечает на сообщения
**Проверьте статус:**
```bash
make status
make health
```

**Возможные причины:**
1. **Неверный токен** - проверьте `TELEGRAM_BOT_TOKEN`
2. **Бот не запущен** - `make start`
3. **Нет интернета** - проверьте подключение
4. **Превышен rate limit** - подождите несколько минут

**Проверка логов:**
```bash
make logs
# Ищите ошибки с ERROR или CRITICAL
```

### Ошибка "Rate limit exceeded"
Превышен лимит запросов к Telegram API. Решения:
1. **Подождите 1-2 минуты** - лимит автоматически сбросится
2. **Уменьшите частоту запросов:**
   ```env
   RATE_LIMIT_PER_MINUTE=10  # вместо 20
   ```
3. **Увеличьте task-периоды** в cron заданиях

### Google Calendar Integration не работает
**Пошаговая диагностика:**
1. **Проверьте credentials.json:**
   ```bash
   ls -la credentials.json
   # Файл должен существовать и быть читаемым
   ```

2. **Тестирование подключения:**
   ```bash
   python test_calendar.py
   ```

3. **Проверьте логи аутентификации:**
   ```bash
   make logs | grep -i "google\|calendar\|auth"
   ```

**Частые ошибки:**
- `401 Unauthorized` - проблемы с credentials
- `403 Forbidden` - Calendar API не включен или нет прав
- `404 Not Found` - неверный Calendar ID

### OCR не распознает текст с изображения
**Проверьте качество изображения:**
- Четкий, контрастный текст
- Хорошее освещение
- Минимальный поворот/наклон
- Размер файла до 10MB

**Поддерживаемые языки:**
```bash
# Проверка установленных языков Tesseract
tesseract --list-langs
```

Если нужного языка нет:
```bash
# Ubuntu/Debian
sudo apt install tesseract-ocr-fra  # французский
sudo apt install tesseract-ocr-spa  # испанский
```

### Высокое потребление памяти
**Мониторинг ресурсов:**
```bash
docker stats telegram-calendar-bot
```

**Оптимизация:**
1. **Уменьшите модель Whisper:**
   ```env
   SPEECH_MODEL=tiny  # вместо base
   ```

2. **Ограничьте размер логов:**
   ```yaml
   # в docker-compose.yml
   logging:
     options:
       max-size: "10m"
       max-file: "3"
   ```

3. **Настройте лимиты контейнера:**
   ```yaml
   deploy:
     resources:
       limits:
         memory: 512M
   ```

## ⚡ Производительность

### Сколько пользователей может обслуживать бот?
При стандартной конфигурации (1 CPU, 512MB RAM):
- **Легкое использование**: 50-100 пользователей
- **Активное использование**: 20-30 пользователей  
- **Интенсивное использование**: 5-10 пользователей

Для масштабирования рекомендуется:
- Увеличить ресурсы сервера
- Оптимизировать модель Whisper (tiny/base)
- Внедрить очереди для обработки файлов

### Как ускорить распознавание речи?
1. **Используйте более быструю модель:**
   ```env
   SPEECH_MODEL=tiny     # Самая быстрая
   SPEECH_MODEL=base     # Хороший баланс (по умолчанию)
   ```

2. **GPU ускорение** (если доступно):
   ```bash
   pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
   ```

3. **Кэширование моделей Whisper:**
   ```bash
   # Предварительная загрузка
   python -c "import whisper; whisper.load_model('base')"
   ```

### Как оптимизировать размер Docker образа?
Текущий образ оптимизирован:
- Многоступенчатая сборка
- Alpine base image для runtime
- Удаление build-зависимостей
- Сжатие слоев

Для дополнительной оптимизации:
```dockerfile
# Используйте distroless образ
FROM gcr.io/distroless/python3-debian11

# Или минимальную Python установку
FROM python:3.11-slim
```

### Производительность разных компонентов

### Возможно ли использовать другие календарные сервисы?
В текущей версии поддерживается только Google Calendar. 

**Планируемая поддержка:**
- Microsoft Outlook/Office365
- Apple iCloud Calendar  
- CalDAV серверы (Nextcloud, etc.)

Для кастомной интеграции можно модифицировать `CalendarService` класс.

### Как добавить поддержку нового языка?
1. **Для NLP парсинга** - добавьте паттерны в `nlp_service.py`:
   ```python
   # Добавьте языковые паттерны
   LANGUAGE_PATTERNS = {
       "fr": {  # французский
           "tomorrow": ["demain"],
           "time": [r"\b(\d{1,2})[h:](\d{2})\b"]
       }
   }
   ```

2. **Для OCR** - установите языковой пакет Tesseract:
   ```bash
   sudo apt install tesseract-ocr-fra
   ```

3. **Для Speech** - Whisper поддерживает 99+ языков автоматически
